Features

Everything you need, nothing you don't

A complete wiki platform with Markdown editing, access control, version history, and team collaboration — all from a single Python process.

📝

Rich Markdown Editor

Split-pane live preview, drag-to-resize divider, formatting toolbar, `[TOC]` table of contents, and fenced code blocks. Draft autosave keeps work safe between sessions.

🗂️

Hierarchical Categories

Unlimited nesting depth with a collapsible tree sidebar. Drag pages and categories to reorder them, and navigate chapters sequentially with Prev / Next buttons.

🔐

Four-Tier Access Control

Roles from read-only user to protected admin. Invite-code signup, per-category editor restrictions, and a lockdown mode that instantly blocks all non-admin access.

🕒

Full Version History

Every save is a versioned snapshot. View any past state, read edit summaries, compare diffs, and revert with a single click — nothing is ever deleted from history.

👥

User Profiles & Badges

Public profiles with avatars, bios, and a GitHub-style contribution heatmap. Auto-trigger badge awards celebrate milestones and keep the team engaged.

💬

Team Messaging

Real-time-style direct messages and group chats built right in — no third-party integration required. Automatic scheduled cleanup keeps storage tidy.

📦

Full Site Migration

Export or import the entire wiki as a ZIP with three conflict modes: delete all, override, or keep existing data. Telegram backup sends DB snapshots automatically.

♿

Accessibility Panel

Per-user preferences for font scale, high-contrast levels, line / letter spacing, reduce-motion, and six color overrides — accessible on every page including the editor.

🔒

Security First

CSRF protection on every form, HTML sanitization after every Markdown render, login rate limiting, security headers on all responses, and constant-time login checks.

Architecture

Simple by design

BananaWiki is a standard Python web app — no build steps, no microservices, no container orchestration required. One process, one database file.

1

Flask + Jinja2

Server-side rendered pages with Jinja2 templates keep the stack simple and fast. No JavaScript framework or build pipeline needed.

2

SQLite database

All wiki pages, users, history, chats, badges, and settings live in a single wiki.db file — trivial to backup, move, or inspect.

3

Gunicorn in production

Serve with Gunicorn behind nginx for production deployments. The included install.sh configures systemd, nginx, and optional SSL automatically.

4

Zero external services

No Redis, no Celery, no S3. Rate-limiting state lives in SQLite, background tasks are lightweight threads, and file uploads are stored on disk.

Installation

Up and running in minutes

Choose the workflow that fits — a one-liner for local development, or an automated script for a production server.

Local development Quickest

# Clone and start with one command
git clone https://github.com/ovtdadt/BananaWiki.git
cd BananaWiki
make dev

# Opens at http://localhost:5001

Automatically creates a virtual environment, installs dependencies, and starts the Flask dev server. Python 3.9+ required.

Production server Recommended

# Automated install (systemd + nginx + SSL)
git clone https://github.com/ovtdadt/BananaWiki.git
cd BananaWiki
sudo make install

# Interactive wizard handles everything

Configures systemd service, nginx reverse proxy, and optionally obtains a Let's Encrypt SSL certificate.

First run setup

On your first visit BananaWiki shows a one-time setup wizard where you create the initial admin account and configure the site name. After that, invite your team via generated invite codes from the admin panel.

Ready to own your wiki?

Clone the repo, run one command, and your team's private knowledge base is live. No accounts, no subscriptions, no data leaving your server.
